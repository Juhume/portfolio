02

Caso: Espejo

"Proyecto end-to-end: desde el diseno tecnico hasta el despliegue."

Problema

Las apps de journaling priorizan rachas y gamificacion. Eso genera presion, no introspeccion.

Los usuarios escriben para mantener una racha, no para reflexionar.

Decisiones
  • Privacidad por defecto, no como opcion avanzada
  • Cifrado E2E real, aunque complique el desarrollo
  • Interfaz silenciosa, sin estimulos innecesarios
  • Funciones solo si no rompen el ritual de escribir
Renuncias

Tan importante como lo que construyes es lo que decides no hacer:

No hay feed
No hay social
No hay algoritmos
No hay growth hacks
Stack

Frontend en Next.js, backend en Node.js, cifrado en cliente, despliegue con Docker.

Next.js React Node.js TypeScript Docker E2E Encryption
Aprendizaje

Tomar decisiones tecnicas completas, no solo escribir codigo.

Disenar la arquitectura, elegir el stack, implementar frontend y backend, configurar el despliegue. Ver el sistema completo.

Enfoque

Proyecto pensado como producto real, con codigo que puede evolucionar. No solo "que funcione", sino que se entienda y se mantenga.

03

Sobre mi

Trabajo en aplicaciones de banca utilizadas por clientes empresa, donde el codigo esta en produccion, los errores importan y los cambios se hacen con cabeza.

Mi dia a dia incluye resolver incidencias tecnicas complejas: autenticacion, estado de aplicacion, rendimiento, conectividad. Trabajo con infraestructuras reales, gestion de trafico con Akamai, sistemas Legacy conviviendo con plataformas NextGen.

Esta experiencia influye en como programo: menos magia, mas claridad; menos promesas, mas fiabilidad.

No me interesa el codigo como fin en si mismo, sino como herramienta para crear sistemas claros, robustos y bien pensados.
